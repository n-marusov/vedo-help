### Review API Contract Completeness
**Source**: 2026-07-05-14.15.md, 2026-07-10-12.13.md, 2026-07-10-16.02.md
**Rule**: During review, flag:
- POST endpoints that create resources without an optional `existing_<entity>_id` field for resend/idempotent scenarios
- Missing coordination contracts when two different endpoints write to the same entity table in the same scenario
- Optional handler modes (create vs reuse) not expressed in the request schema
- Admin API routes not registered in `admin_router` or missing explicit RBAC middleware
- Dynamic SQL building with hardcoded `$N` parameter numbering where conditional parameters may leave gaps
