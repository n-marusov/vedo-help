## Problem

The web crawler feature was implemented as a CRUD scaffold without the actual crawling engine. The plan specified 10 tasks, but core functionality was missing:

- No BFS crawler (WebCrawler struct, ContentExtractor, RobotsChecker)
- No scraper/robotstxt dependencies in Cargo.toml
- No background crawl pipeline (chunking, embedding, Chroma indexing)
- No SSE subscription endpoint for real-time progress
- No retry endpoint for failed pages
- Missing migration 17 (add 'web' source to documents constraint)
- ChunkSearchParams.source didn't include 'web'

## Root Cause

The implementation was scoped to basic CRUD operations (DB schema, models, repository, UI component, basic API handlers) but the actual crawling functionality was not developed. This left the module as a shell that could create/manage crawl jobs but never execute them.

## Solution

1. **Rewrite crawler.rs** with `WebCrawler` struct using `reqwest::Client`, BFS crawl algorithm, `ContentExtractor` using `scraper` for HTML parsing with heading-to-markdown conversion, and `RobotsChecker` for robots.txt parsing/caching.

2. **Expand service.rs** with a background crawl pipeline: `start_crawl` spawns `tokio::spawn` tasks that run BFS crawl → save documents → chunk → embed via RouterAI → index into Chroma. Added `cancel_signals` HashMap with broadcast channels for job cancellation and `crawl_progress` for polling-based progress tracking.

3. **Add handlers** for `retry_failed` (POST /api/web-crawl/:id/retry) and `subscribe` (GET /api/web-crawl/:id/subscribe) using SSE with polling-based stream.

4. **Add migration 17** to include 'web' in documents source constraint.

5. **Update Cargo.toml** with `scraper 0.22` and `robotstxt 0.3`.

6. **Update frontend types** to include 'web' in ChunkSearchParams.source.

## Prevention

- When implementing modules with an engine component, ensure the core algorithm (BFS crawl, in this case) is built before wiring CRUD scaffolding
- Verify all plan dependencies are added to Cargo.toml before marking the task complete
- Check that related type changes (like ChunkSearchParams.source) are updated across the stack
- Run `cargo check` to verify compilation after backend changes
