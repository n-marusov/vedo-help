# Empty Session Debug List on Initial Admin Panel Load

**Date:** 2026-07-05 11:20
**Files:** frontend/src/components/SessionDebug.vue, frontend/src/components/__tests__/SessionDebug.spec.ts
**Severity:** medium

## Problem

The Session Debug panel in the admin interface showed an empty list ("No sessions found") when first loaded, even though sessions existed in the database. The user had to type something in the search field to trigger a search. With all filters reset, the list remained empty until manual interaction.

## Root Cause

The `SessionDebug.vue` component initialized `sessions` as an empty array but never triggered the initial data load on mount. There was no `onMounted` hook that called `searchSessions()`. The component relied solely on user interaction (`@input` events on filter fields) to fetch data. On first render with empty filters, `sessions.length === 0 && !isLoading` evaluated to `true`, showing the "No sessions found" empty state.

The test also confirmed this: the test `renders session list after search` explicitly filled the search input before expecting results, masking the missing initial load.

## Solution

Added `onMounted` from `vue` and a `searchSessions()` call inside it:

```
import { computed, onMounted, ref } from 'vue';

onMounted(() => {
  searchSessions();
});
```

When `searchSessions()` is called with all filters empty (`''`), the API client sends `GET /admin/sessions` with no query parameters, and the backend runs `SELECT ... FROM sessions WHERE 1=1` — returning all sessions. This matches the expected behavior.

Also added a new test `'loads all sessions on mount'` that verifies:
- `adminSearchSessions` is called exactly once on mount
- It is called with all params as `undefined` (no filters)
- Two session items are rendered

## Prevention

- Any Vue component that displays a data list should load data on mount via `onMounted` unless there is a deliberate reason to wait for user interaction.
- Tests should include a baseline "loads data on mount" case to catch missing initial load patterns.
- Review existing admin panel components for similar missing `onMounted` patterns.
