# Fix FK Constraint Violations and Race Conditions in Document Service Tests

**Date:** 2026-06-21 02:40
**Files:** `backend/src/modules/documents/service.rs`, `backend/src/modules/documents/repository.rs`, `backend/Cargo.toml`
**Severity:** high

## Problem

11 out of 56 backend unit tests failed with `foreign key constraint "documents_collection_id_fkey"` when running `cargo test --lib` with default parallel execution. 7 tests called `process_upload`/`process_zip_upload` with a random `collection_id` but never inserted a parent row into `collections`, causing FK violation on `INSERT INTO documents`. 4 tests that did insert collections still failed because `make_service()` called `TRUNCATE TABLE … CASCADE`, which — under parallel test execution — deleted collections inserted by other tests (race condition).

Additionally, `test_reload_document_deactivates_old_chunks_and_saves_new_active_chunks` had a secondary bug: `sqlx::query_as` decoded the `"index"` column (PostgreSQL `INTEGER`/INT4) as Rust `i64` (INT8), causing a type mismatch at runtime.

## Root Cause

1. **Missing parent rows**: Tests using `process_upload` and `process_zip_upload` generated a `Uuid::new_v4()` for `collection_id` but never inserted the corresponding row into `collections`, violating FK `documents_collection_id_fkey`.

2. **TRUNCATE race condition**: `make_service()` truncated all tables on every call. Under parallel test execution with `cargo test`'s default multi-threaded runner, one test's `TRUNCATE` erased data created by another running test.

3. **INT4/i64 type mismatch**: The `chunks.index` column is PostgreSQL `INTEGER` (INT4), but `get_chunks` and `get_active_chunks` decoded it as `i64` (INT8), and `save_chunk` bound it as `i64`. This caused `mismatched types` errors at runtime when sqlx performed strict type checking.

4. **Raw string escape bug**: A test query used `\"index\"` inside an `r#"..."#` raw string, producing `\"index\"` as literal text instead of the intended `"index"`, causing a PostgreSQL syntax error.

## Solution

- Added `INSERT INTO collections` before every `process_upload`/`process_zip_upload` call in tests, with unique names incorporating the `collection_id` UUID to avoid PK and UNIQUE constraint collisions.
- Kept `TRUNCATE TABLE … CASCADE` in `make_service()` for test isolation, but added `serial_test` crate with `#[serial]` attribute on all document service tests to enforce sequential execution, eliminating race conditions.
- Changed `i64` to `i32` for the `"index"` column in `save_chunk`, `get_chunks`, and `get_active_chunks` to match the PostgreSQL `INTEGER` type.
- Fixed the raw string SQL query in `test_reload_document` by removing unnecessary backslash escaping.
- Replaced `.ok()` with `.expect("Failed to insert collection")` on all test collection inserts to prevent silent failures.

## Prevention

- Always insert parent rows before referencing FK columns in test data. Use `format!("test-name-{collection_id}")` for unique names.
- When database tests share a single database, use `#[serial]` (from `serial_test`) or `--test-threads=1` to prevent race conditions from `TRUNCATE`/`INSERT` interleaving.
- Match PostgreSQL column types exactly in sqlx query_as tuples: `INTEGER` → `i32`, `BIGINT` → `i64`. Never assume a wider Rust type works for a narrower SQL type.
- Use `.expect()` instead of `.ok()` for test setup operations — silent failures mask FK violations and make debugging harder.
