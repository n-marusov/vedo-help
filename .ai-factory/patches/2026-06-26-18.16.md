# Sidebar starts collapsed instead of expanded matching design

**Date:** 2026-06-26 18:16
**Files:** frontend/src/stores/chat.ts, frontend/src/views/ChatView.vue, frontend/src/components/__tests__/ChatWindow.spec.ts
**Severity:** medium

## Problem

The chat page sidebar was displayed collapsed (48px narrow vertical strip) instead of expanded (312px) as shown in the design (chat.pen). In the collapsed state, only the search button was visible — the session title, new session button, and session list were hidden. The collapse/expand button always showed the same chevron-left icon and "Collapse sidebar" title regardless of state, with no CSS transition for the width change.

## Root Cause

The `sidebarCollapsed` ref in the chat store initialized from `localStorage.getItem('chat_sidebar_collapsed') === 'true'`. If localStorage had this key set to `'true'` (e.g., from a previous testing session or dev tool interaction), the sidebar would start collapsed on page load, contrary to the design which shows the sidebar always expanded. Additionally, the collapse/expand button was not reactive to the state — it always rendered a chevron-left icon and "Collapse sidebar" tooltip even when the sidebar was already collapsed.

## Solution

1. **Changed default to always expanded**: Set `sidebarCollapsed` initial value to `false` unconditionally. The toggle still persists to localStorage so the preference survives page refresh, but a fresh page load always starts expanded matching the design.

2. **Made collapse button reactive**: Added `v-if` / `v-else` for the SVG path to show chevron-left (◀) when expanded and chevron-right (▶) when collapsed. The `title` attribute dynamically changes between "Collapse sidebar" and "Expand sidebar".

3. **Added CSS transition**: Added `transition` for `width`, `min-width`, and `padding` to `.session-sidebar` for smooth collapse/expand animation.

4. **Added regression tests**: Added 7 new tests to ChatWindow.spec.ts covering: sidebar starts expanded by default, session title is visible, all header controls render, session list renders when sessions exist, collapsed class toggles correctly, and collapse button title changes based on state.

## Prevention

- UI components that have persisted state should always verify the default state matches the design spec, not just a localStorage value
- Toggle buttons should always visually reflect the current state (icon, title) to avoid confusing the user
- Add regression tests for initial component state to catch persistence-related regressions
- CSS transitions should be considered for any width/visibility toggles to provide smooth UX
