# Decode Git repository UUIDs stored as SQLite TEXT

**Date:** 2026-06-19 01:30
**Files:** backend/src/modules/git_sync/repository.rs
**Severity:** high

## Problem

Creating a Git repository succeeded at the `INSERT` step, but the endpoint then
returned an internal error while fetching the newly-created summary:

`Database error: error occurred while decoding column "id": invalid length: expected 16 bytes, found 36`

The failure happened for `get_repo_with_collection_name()` after inserting a
repository linked to a collection with a Cyrillic display name.

## Root Cause

The `git_repositories` table stores UUIDs as SQLite `TEXT` values, using the
canonical 36-character UUID string. `GitRepo` and `GitRepoSummary` derived
`sqlx::FromRow` with `uuid::Uuid` fields, so `sqlx` attempted to decode those
columns directly as SQLite UUID values and expected a 16-byte UUID blob.

Other repositories in the project already avoid this by reading UUID columns as
`String` and parsing them with `Uuid::parse_str()`. Git sync did not follow that
SQLite TEXT UUID pattern.

## Solution

Added internal row DTOs for Git repository queries:

- `GitRepoRow`
- `GitRepoSummaryRow`

These row types decode database UUID and timestamp fields as strings, then
convert them into domain/API models via `TryFrom`. Parsing failures now produce
structured `AppError::InternalError` values and emit logs with the
`[FIX:git-repo-uuid-text]` prefix.

Added a regression test that inserts a Git repository into an in-memory SQLite
schema using TEXT UUIDs, then verifies `get_repo_with_collection_name()` returns
a valid `GitRepoSummary` with the original UUIDs and Cyrillic collection name.

## Prevention

- For SQLite tables that store UUIDs as `TEXT`, do not derive `sqlx::FromRow`
  directly into structs containing `uuid::Uuid` fields.
- Use row DTOs with string fields and explicit parsing into domain models.
- Add repository-level regression tests whenever a query maps SQLite rows into
  UUID-rich domain structs.
- During review, check that `bind(uuid.to_string())` is paired with
  `Uuid::parse_str()` on read paths.
