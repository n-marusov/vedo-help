# Position teleported VSelect dropdown under trigger

**Date:** 2026-06-19 01:45
**Files:** frontend/src/components/ui/VSelect.vue, frontend/src/components/__tests__/ChatWindow.spec.ts
**Severity:** medium

## Problem

Clicking the `Select a collection...` dropdown in the main chat widget did not
show existing collections to the user, even when collections were loaded.

## Root Cause

`VSelect` teleports its dropdown to `body`, but the dropdown was styled with
`position: absolute` and only received `minWidth`. After teleporting, it no
longer had the select component as its positioning context, so the dropdown
could render away from the trigger or outside the visible chat toolbar area.

## Solution

`VSelect` now calculates the trigger's viewport rectangle when opened and stores
explicit dropdown coordinates:

- `left` from `trigger.getBoundingClientRect().left`
- `top` from `trigger.getBoundingClientRect().bottom + 4`
- `minWidth` from the trigger width

The teleported dropdown uses `position: fixed`, so these viewport coordinates
place it directly under the select trigger. Position is refreshed on resize,
scroll, and option changes while open. Added `[FIX:v-select-dropdown]` debug
logging when the dropdown is positioned.

Added a regression test for `ChatView` that mocks loaded collections, opens the
chat collection selector, and verifies the collection option text is present in
the teleported dropdown.

## Prevention

- When using `Teleport to="body"`, do not rely on parent-relative absolute
  positioning.
- Position overlays using viewport coordinates (`fixed`) or a dedicated floating
  UI helper.
- Add tests for teleported dropdowns that assert visible option content after
  user interaction.
