## Problem

314 E2E tests failed across 8 root causes when running `npx playwright test` (3 projects: chromium, mobile, tablet). After fixing all 8 root causes in chromium: **145 passed, 0 failed**.

## Root Causes

1. **WebKit not installed** (290 failures): `npx playwright install webkit` was never run, causing all mobile/tablet tests to fail with "Executable doesn't exist".

2. **Chat input disabled** (6 failures): `chat-input` has `:disabled="!activeCollectionId"`. Tests for Enter/Shift+Enter/message animations didn't set `activeCollectionId` in the Pinia store before interacting with the input.

3. **ZIP upload DOM timing** (4 failures): `input[type="file"]` is hidden (`display: none`) inside VDropZone, which itself is conditionally rendered via `v-if="activeCollectionId"`. Tests didn't wait for VDropZone render after setting collection. Also mocked wrong URL (`/api/documents/upload` instead of `/api/documents/upload-zip` — ZIP uploads use XMLHttpRequest).

4. **auth-card not found** (3 failures): With valid JWT, admin page shows `admin-panel` not `auth-card`. Tests checked for `auth-card`/`auth-title` which don't exist.

5. **VSelect native option** (1 failure): Test checked for native `<option>` elements, but VSelect renders a custom dropdown (Teleported to body) with `<button>` options.

6. **Git sync assertions** (3 failures): Status regex too strict (`/idle/i` only — can briefly show "syncing"); 401 redirect checks specific elements instead of URL; empty state regex had Russian locale.

7. **NDJSON streaming** (2 failures): `ReadableStream` in `route.fulfill()` not processed correctly — replaced with plain string body.

8. **CSS assertions** (6 failures): `expect.stringContaining('rgb')` instead of `/rgb/i` regex; missing `flex-direction: column` at 480px breakpoint; toolbar/composer background not themed.

## Solution

- Installed WebKit via `npx playwright install webkit`
- Added `activeCollectionId` setup in affected tests before fill/interactions
- Changed mock URLs to `/api/documents/upload-zip`, added activeCollectionId + waitForSelector('.drop-zone') in zip tests
- Replaced auth-card assertions with admin-view assertions for authenticated state
- Updated VSelect test to click trigger → wait for dropdown → count button options
- Fixed git-sync: relaxed regex, used `addInitScript` override for unauth test, fixed English text
- Replaced ReadableStream with joined string body in rag-flow mocks
- Fixed CSS assertion format to use regex, added missing media query and toolbar tokens

## Prevention

- When adding `:disabled` conditions to form controls, audit all E2E tests that interact with them
- Always mock the exact endpoint URL — check the store/component for the real URL
- Use `page.waitForSelector()` on conditionally rendered elements
- For auth-dependent tests, check what the authenticated UI actually shows vs unauthenticated
- For custom components (VSelect), test with the component's actual DOM structure
- Use `page.route().fulfill()` with string body instead of ReadableStream for mock responses
- `borderColor` shorthand may return unexpected values; use longhand (`borderTopColor`) or check `not.toBe()`
