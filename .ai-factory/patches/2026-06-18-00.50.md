# Hardcoded API proxy target in Vite config causes ECONNREFUSED in Docker Compose

**Date:** 2026-06-18 00:50
**Files:** frontend/vite.config.ts, docker-compose.override.yml
**Severity:** high

## Problem

Vite dev server proxied `/api/*` requests to a hardcoded `http://localhost:3000`.
When running in Docker Compose, the frontend container tried to connect to
`localhost:3000` (its own container) instead of the `backend` service,
resulting in `AggregateError [ECONNREFUSED]` for all API calls.

Error from frontend console:
```
[vite] http proxy error: /api/collections
AggregateError [ECONNREFUSED]:
  at internalConnectMultiple (node:net:1134:18)
  at afterConnectMultiple (node:net:1715:7)
```

## Root Cause

The `/api` proxy target in `vite.config.ts` was hardcoded as a string literal
(`'http://localhost:3000'`) instead of being configurable via environment
variable. While the `/auth` proxy already used `VITE_KEYCLOAK_PROXY_TARGET`,
the `/api` proxy was overlooked when the project adopted environment-based
configuration.

Additionally, `docker-compose.override.yml` set `VITE_KEYCLOAK_PROXY_TARGET`
for the frontend service but did not set the corresponding variable for the
API proxy, so the Docker container fell back to the default `localhost:3000`.

## Solution

1. **`frontend/vite.config.ts`:** Added `apiProxyTarget` variable
   (`VITE_API_PROXY_TARGET` env var with fallback to `http://localhost:3000`)
   and used it as the `/api` proxy target.

2. **`docker-compose.override.yml`:** Added `VITE_API_PROXY_TARGET:
   http://backend:3000` to the frontend service's `environment` block,
   matching the pattern already used for `VITE_KEYCLOAK_PROXY_TARGET`.

This ensures:
- Local dev without Docker → fallback to `http://localhost:3000` (unchanged)
- Docker Compose → frontend proxies to `backend:3000` over internal network
- Custom setup → override via `VITE_API_PROXY_TARGET` env var

## Prevention

- When adding proxy targets to Vite config, always use environment variables
  with sensible defaults rather than hardcoded strings.
- Ensure all proxy targets have corresponding overrides in
  `docker-compose.override.yml` for the frontend service's environment.
- Code review checklist: "Is every hostname/port in config configurable via
  env var?"
