# Fix E2E test failures: ZIP validation, Chroma retry, skeleton tests, timeouts

**Date:** 2026-06-21 14:55
**Files:**
- `backend/src/modules/documents/service.rs`
- `backend/src/modules/query/service.rs`
- `frontend/e2e/loading-skeletons.spec.ts`
- `frontend/e2e/rag-flow.spec.ts`
- `frontend/e2e/chat-export.spec.ts`
**Severity:** medium

## Problem

Five E2E test failures caused by:

1. **RAG-003 / chat-export:** Queries return 0 results because Chroma propagation delay (filter `is_active`) hasn't replicated the newly indexed chunks within ~400ms.
2. **ZIP upload 413 test:** Playwright multipart sends a ZIP that `validate_zip_magic` rejects per-file before the 10-file limit check runs, returning HTTP 415 instead of 413.
3. **Loading skeleton tests:** `page.route` registered *after* the API call was already dispatched (parallel test timing), so the intercept never fired.
4. **Git-repos skeleton:** `setActiveCollection` triggered `GitRepoManager` mount + API call before the route was registered.
5. **Chat-export:** Timeout (15s) too tight for LLM streaming + Chroma retries.

## Root Cause

1. **ZIP validation order:** The 10-file limit check ran *after* per-file `validate_file`, meaning a malformed ZIP entry failed with 415 before the size check could return 413.
2. **Chroma eventual consistency:** Chroma's `is_active` filter propagation can take >400ms post-index. No retry loop existed for the first query.
3. **Skeleton test race condition:** `page.route({times:1})` registered too late — the API request fired during `goto` or `setActiveCollection` before the route handler was active.
4. **Route ordering in git-sync test:** `setActiveCollection` (which triggers GitRepoManager) called before route registration.
5. **Insufficient timeout:** LLM streaming through OpenRouter + Chroma retry loop can exceed 15s.

## Solution

1. **ZIP validation:** Added early `archive.len() > 10` check immediately after `ZipArchive::new()`, before any per-file extraction. Removed duplicate check after extraction. Logged with `tracing::info!` with `[FIX]` context.
2. **Chroma retry:** Replaced single empty-result check with up to 3 retries, 500ms apart, in `process_query`. Results found after retry are logged. Made `chroma_results` `mut`.
3. **Skeleton routes:** Moved `page.route()` calls before `page.goto()` / `setActiveCollection` / git-tab click in all tests.
4. **Git-sync route order:** Register route *before* `setActiveCollection`, then evaluate the Pinia state change.
5. **Timeouts:** Increased assistant message `waitForSelector` from 15s to 20s in both chat-export tests. Added 2s Chroma propagation pause after document upload in `TC-RAG-003`.
6. **Total files fix:** Removed `file_count` variable that was deleted with the duplicate check; replaced with `processed + failed`.

## Prevention

- Always order validation: cheap/global checks (entry count, size limits) before per-item validation.
- Add retry loops for services with known eventual consistency (Chroma, Elasticsearch).
- In Playwright tests, register route handlers before any action that could trigger the request.
- For LLM-dependent tests, allow ample timeouts (20-30s) and add propagation waits after data mutations.
- When removing variables, verify all references are updated — the `file_count` removal broke the response struct.
