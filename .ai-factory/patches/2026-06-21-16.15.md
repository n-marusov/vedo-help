# PostgreSQL uuid = text type mismatch in get_chunks_by_ids

**Date:** 2026-06-21 16:15
**Files:** `backend/src/modules/query/repository.rs`, `backend/tests/integration.rs`
**Severity:** high

## Problem

Querying a collection with chunks returned `"Internal error: Failed to fetch chunks: error returned from database: operator does not exist: uuid = text"`.

The RAG pipeline failed because `get_chunks_by_ids` received chunk IDs from Chroma as `Vec<String>` (valid UUID strings), but sqlx bound them as `text` to the PostgreSQL `WHERE c.id IN (...)` clause. `chunks.id` is `UUID`, and PostgreSQL does not implicitly cast `text` to `uuid` inside `IN` clauses.

## Root Cause

The method `get_chunks_by_ids` in `query/repository.rs` accepted `&[String]` and bound each string directly via `query.bind(id)`. sqlx's PostgreSQL driver sends text parameters as-is. Since the column is `UUID`, PostgreSQL raises `operator does not exist: uuid = text` — no implicit cast exists for the equality operator between these two types.

This was introduced when the query module was built: Chroma returns chunk IDs as opaque strings (which happen to be UUIDs formatted with hyphens), and the code assumed sqlx+PostgreSQL would coerce them automatically.

## Solution

Parse each `String` into `uuid::Uuid` before binding:

```rust
let uuid = Uuid::parse_str(id).map_err(|e| {
    AppError::InternalError(format!("Invalid chunk UUID from Chroma: {e}"))
})?;
query = query.bind(uuid);
```

Also changed the `query_as` tuple type from `(String, ...)` to `(uuid::Uuid, ...)` and the `HashMap` key from `String` to `Uuid` — using a typed key is both correct and slightly faster.

## Prevention

- Any time a `String` parameter is bound to a PostgreSQL `UUID` column, parse it to `uuid::Uuid` first. PostgreSQL has no implicit `text → uuid` cast in `=` / `IN` comparisons.
- Add repository-level regression tests that exercise `get_chunks_by_ids` with string UUID inputs (exactly the public API used by the query service).
- Consider a clippy lint or sqlx compile-time check for `bind(String)` against `UUID` columns (though sqlx compile-time checks may not catch dynamic queries built with format! placeholders).
