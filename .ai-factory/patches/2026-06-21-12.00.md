# FastAPI on_event deprecation and TestClient httpx2 warning

**Date:** 2026-06-21 12:00
**Files:** embedding/src/main.py, embedding/tests/conftest.py, embedding/tests/test_main.py
**Severity:** low

## Problem

Two deprecation warnings in the embedding service:

1. `DeprecationWarning: on_event is deprecated, use lifespan event handlers instead` from `@app.on_event("startup")` in `main.py`
2. `StarletteDeprecationWarning: Using httpx with starlette.testclient is deprecated; install httpx2 instead` from `TestClient` import in test files

## Root Cause

- `@app.on_event("startup")` is the old FastAPI startup pattern. FastAPI 0.115+ recommends using a `lifespan` async context manager passed to `FastAPI(lifespan=...)`. The old `on_event` decorator triggers deprecation warnings and will be removed in a future version.
- `starlette.testclient.TestClient` wraps `httpx` and Starlette is deprecating this in favor of using `httpx.AsyncClient` with `httpx.ASGITransport` directly. This gives more control over async test lifecycle and avoids the deprecated wrapper.

## Solution

- Replaced `@app.on_event("startup")` with a `lifespan` async generator function. The lifespan function configures logging and initializes the `embedder` global before `yield`, then the `yield` signals the app is ready. The `FastAPI` app is created with `lifespan=lifespan`.
- Replaced `TestClient` in test fixtures with `httpx.AsyncClient` using `httpx.ASGITransport(app=app)`. Since `ASGITransport` does not automatically run lifespan events, the fixture manually initializes `main_mod.embedder = CachedEmbedder()` and cleans up in `finally`.
- Converted test methods from sync to `async` and changed `test_client.get/post` calls to `await test_client.get/post`.

## Prevention

- Prefer `lifespan` over `@app.on_event` for all new FastAPI apps — it's the officially recommended pattern and supports startup/shutdown in one place.
- Use `httpx.AsyncClient` with `ASGITransport` for testing async FastAPI apps instead of `starlette.testclient.TestClient`.
- When switching from `TestClient` to `httpx.AsyncClient`, remember that lifespan startup/shutdown must be handled explicitly in the test fixture since `ASGITransport` does not invoke lifespan events automatically.
