# Pipeline stage hidden when recovery placeholder is replaced

**Date:** 2026-07-10 01:18
**Files:** frontend/src/views/ChatView.vue, frontend/src/components/__tests__/ChatWindow.spec.ts
**Severity:** medium

## Problem

After a browser reload during an in-flight RAG query, the session title was restored and backend recovery SSE emitted `pipeline_stage` events, but the pipeline stage was still not visible in the UI.

## Root Cause

The visual stage indicator was rendered only inside `MessageBubble` when the last message was an empty assistant streaming placeholder. During recovery, the store can briefly or permanently have persisted/user messages without a visible empty assistant placeholder, so `pipelineStageLabel` was updated correctly but had no reliable render target.

## Solution

Added a ChatView-level fallback pipeline status bar that is shown when `chatStore.isLoading` and `chatStore.pipelineStageLabel` are set, but no visible empty assistant streaming placeholder is present. This keeps the existing bubble indicator for normal streaming while making recovery state visible after reload even when the message list shape changes.

A regression test now verifies that the fallback renders `Reranking results...` when the pipeline is active and no streaming placeholder is visible.

## Prevention

- Do not bind critical recovery UI exclusively to temporary placeholder messages.
- Recovery state indicators should have a stable render target independent of message list reconciliation.
- UI tests for reload recovery should assert visible pipeline stage text, not only restored session title or collection badge.
