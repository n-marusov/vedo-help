# Update pnpm to 11.9.0

**Date:** 2026-07-04 12:00
**Files:** frontend/Dockerfile, frontend/package.json, .github/workflows/e2e.yml
**Severity:** low

## Problem

pnpm install outputs a warning about an available update from 10.34.4 to 11.9.0, recommending to run `corepack use pnpm@11.9.0`.

## Root Cause

The `frontend/Dockerfile` and GitHub Actions workflow explicitly pinned the pnpm version via `corepack prepare pnpm@10 --activate` (or lacked the specific version and used a default). The `frontend/package.json` was missing the `packageManager` field which `corepack` uses to resolve the correct package manager version.

## Solution

- Updated `frontend/Dockerfile` to use `corepack prepare pnpm@11.9.0 --activate` across all Node.js stages.
- Added `"packageManager": "pnpm@11.9.0"` to `frontend/package.json`.
- Updated `.github/workflows/e2e.yml` to explicitly use `corepack prepare pnpm@11.9.0 --activate` alongside `corepack enable`.

## Prevention

- Always pin package manager versions using the `packageManager` field in `package.json`.
- Keep `Dockerfile` `corepack prepare` versions synced with `package.json` to prevent mismatched environments or install-time update warnings.
