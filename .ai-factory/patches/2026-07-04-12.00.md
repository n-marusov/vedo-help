# Update pnpm from 10 to 11.9.0

**Date:** 2026-07-04 12:00
**Files:** frontend/Dockerfile, frontend/package.json, frontend/pnpm-workspace.yaml, frontend/.npmrc, .github/workflows/e2e.yml, .gitignore
**Severity:** low

## Problem

pnpm install outputs a warning about an available update from 10.34.4 to 11.9.0, recommending to run `corepack use pnpm@11.9.0`. Additionally, pnpm v11 blocks build scripts by default, causing `ERR_PNPM_IGNORED_BUILDS` during Docker builds.

## Root Cause

1. Version was pinned to `pnpm@10` in Dockerfile via `corepack prepare pnpm@10 --activate`.
2. `package.json` was missing the `packageManager` field.
3. pnpm v11 moves configuration out of `package.json` — the `pnpm.onlyBuiltDependencies` field is no longer read.
4. Build script approval (`allowBuilds`) must now be configured in `pnpm-workspace.yaml`, not `.npmrc`.
5. `pnpm-workspace.yaml` was in `.gitignore`, preventing it from being committed/deployed.
6. Dockerfile was only copying `package.json` and `pnpm-lock.yaml`, missing the config file.

## Solution

1. Updated Dockerfile: `corepack prepare pnpm@10` → `corepack prepare pnpm@11.9.0` in all stages.
2. Added `"packageManager": "pnpm@11.9.0"` to `package.json` for corepack auto-detection.
3. Created `pnpm-workspace.yaml` with `allowBuilds` entries for packages needing build scripts.
4. Removed `pnpm-workspace.yaml` from `.gitignore`.
5. Updated Dockerfile COPY instructions to include `pnpm-workspace.yaml` alongside manifests.
6. Updated `.github/workflows/e2e.yml` to explicitly prepare pnpm@11.9.0.
7. Removed deprecated `pnpm.onlyBuiltDependencies` from `package.json` (pnpm 11 ignores it).

## Prevention

- Always pin package manager versions using the `packageManager` field in `package.json`.
- In pnpm v11+, configure `allowBuilds` in `pnpm-workspace.yaml` (not in `package.json` or `.npmrc`).
- Ensure `pnpm-workspace.yaml` is NOT in `.gitignore` and IS copied in Dockerfiles.
- Keep `corepack prepare` versions synced with `package.json`.
- After upgrading pnpm major version, run `pnpm approve-builds` to regenerate config.
