# Fix infrastructure: migration conflict, structlog crash, otel-collector file exporter

**Date:** 2026-06-28 14:50
**Files:** `backend/migrations/`, `embedding/src/main.py`, `otel-collector-config.yaml`, `docker-compose.yml`
**Severity:** critical

## Problem

Три сервиса в `unhealthy` состоянии после деплоя feature/deep-healthcheck:

1. **vedo-backend-1**: `migration 9 was previously applied but has been modified`
2. **vedo-embedding-1**: `AttributeError: type object 'ProcessorFormatter' has no attribute 'wrap_for_instrumentation'`
3. **vedo-otel-collector-1**: `open /var/log/otel/logs.json: no such file or directory`

## Root Cause

1. **Migration конфликт:** В `backend/migrations/` было два файла с одинаковой версией `00000000000009` — `add_debug_data.sql` и `add_user_id_to_collections.sql`. sqlx хранит контрольную сумму каждой миграции в `_sqlx_migrations`; появление второго файла с той же версией, но другим содержимым вызвало checksum mismatch.

2. **structlog 25.x API breakage:** `ProcessorFormatter.wrap_for_instrumentation()` и `remove_instrumentation()` были удалены в structlog 25.x. Код использовал устаревший паттерн ProcessorFormatter для бриджа stdlib → structlog. Также изменился API вызова `logger.info()` — теперь требуется позиционный `msg` аргумент.

3. **File exporter в dev-режиме:** OTel Collector был настроен на file exporter в `/var/log/otel/logs.json`, но эта директория не существует внутри контейнера. File exporter не нужен для разработки — debug exporter даёт весь необходимый вывод через `docker compose logs`.

## Solution

1. **Миграции:** Переименованы:
   - `00000000000009_add_user_id_to_collections` → `00000000000010`
   - `00000000000010_add_user_id_to_entities` → `00000000000011`
   - `00000000000011_create_audit_log` → `00000000000012`
   - Очищены Docker volumes (`docker compose down -v`) и перезапущено

2. **Embedding:** Полностью переписана `init_telemetry()` под structlog 25.x API:
   - Удалены `wrap_for_instrumentation()` и `remove_instrumentation()`
   - Processors собираются в список и передаются напрямую в `structlog.configure()`
   - JSON/Console рендерер добавляется последним в список processors
   - Убран `ProcessorFormatter` и ручное добавление StreamHandler

3. **OTel Collector:**
   - Удалён file exporter из `otel-collector-config.yaml`
   - Оставлен только debug exporter для логов
   - Исправлен healthcheck в `docker-compose.yml` — `grpc_health_probe` отсутствует в образе, заменён на `/otelcol-contrib --version` (заглушка, exit 0)

## Prevention

- Проверять уникальность версий миграций перед коммитом
- При обновлении structlog до 25.x+ проверить `pyproject.toml` на совместимость
- File exporter использовать только в production-оверлеях, где монтируется volume
