# Docker-internal service URLs and Chroma naming constraints

**Date:** 2026-06-18 19:20
**Files:** `docker-compose.yml`, `backend/src/modules/collections/service.rs`, `.ai-factory/skill-context/aif-fix/SKILL.md`
**Severity:** high

## Problems

Creating a collection through `POST /api/collections` failed with a 502 response.

**Problem 1 — Docker service discovery:** Backend could not reach Chroma at `http://localhost:8000` because inside the container `localhost` refers to the container itself. Connection refused after 3 retries.

**Problem 2 — Chroma naming constraints:** After fixing the URL, Chroma rejected the Cyrillic name `Техническая документация` with HTTP 400:

```text
Chroma create_collection failed (attempt 1/3): Chroma error: Create collection failed (HTTP 400 Bad Request): {"error":"InvalidArgumentError","message":"Expected collection name that (1) contains 3-63 characters, (2) starts and ends with an alphanumeric character, (3) otherwise contains only alphanumeric characters, underscores or hyphens (-), (4) contains no two consecutive periods (..) and (5) is not a valid IPv4 address, got Техническая документация"}
```

After three retry attempts the backend rolled back the SQLite collection row.

## Root Causes

**Root Cause 1 — Docker service discovery:**

`CHROMA_URL` and `EMBEDDING_SERVICE_URL` were absent from the backend service environment in `docker-compose.yml`, so `backend/src/config.rs` fell back to host-local defaults:

- `CHROMA_URL=http://localhost:8000`
- `EMBEDDING_SERVICE_URL=http://localhost:8001`

That assumption is valid only when the backend process runs directly on the host. Inside Docker Compose, `localhost` points to the backend container itself. Chroma and embedding run as sibling services and must be addressed through Compose DNS names on the internal network.

**Root Cause 2 — Chroma naming constraints:**

`CollectionService::create` passed the user-visible display name (e.g. `Техническая документация`) directly to `chroma.create_collection()`. Chroma enforces a strict naming rule: only ASCII alphanumeric, underscores (`_`), and hyphens (`-`) are allowed. The `QueryService` already used the collection UUID as the Chroma collection name (`request.collection_id.to_string()`), but the collection service wasn't doing the same.

Both are recurring patterns: agents often copy browser/host URLs into container-to-container settings, and pass user-supplied strings to APIs that enforce ASCII naming constraints.

## Solutions

**Solution 1 — Docker service discovery:**

The backend service in `docker-compose.yml` now sets internal URLs explicitly while keeping them overridable:

```yaml
CHROMA_URL: "${CHROMA_URL:-http://chroma:8000}"
EMBEDDING_SERVICE_URL: "${EMBEDDING_SERVICE_URL:-http://embedding:8001}"
```

**Solution 2 — Chroma naming constraints:**

`CollectionService::create` and `CollectionService::delete` now use the collection UUID (`id.to_string()`) as the Chroma collection name instead of the user-supplied display name. The display name is stored only in SQLite and shown in the UI.

A project-specific `/aif-fix` skill context was added at `.ai-factory/skill-context/aif-fix/SKILL.md` to encode both generalized rules for future agents.

## Prevention

### Docker service discovery
- Before changing Docker or service connectivity, classify every URL as browser/public, host-local, or Docker-internal.
- Use Compose service names plus container ports for all container-to-container calls.
- Keep `localhost` only for browser-facing URLs, host-published ports, self-healthchecks, and host-run tests.
- In reviews, inspect environment variables ending in `_URL`, `_URI`, `_HOST`, and proxy targets for scope mismatches.
- Add a regression check that renders Compose config and verifies backend environment contains `CHROMA_URL=http://chroma:8000` and `EMBEDDING_SERVICE_URL=http://embedding:8001`.

### Chroma naming
- Never pass user-supplied names directly to Chroma API — Chroma accepts only ASCII alphanumeric, underscores, and hyphens.
- Use a deterministic surrogate (UUID, slugified ID) as the Chroma collection name.
- Store the user-visible display name in SQLite; query Chroma by the deterministic key.
- When reading code, verify that every `chroma.create_collection()` or `chroma.delete_collection()` call uses a safe identifier, not user input.
- Test collection creation with non-ASCII characters (Cyrillic, CJK, accented Latin) during integration testing.

### General
- When connecting a backend to an external API that enforces naming constraints, identify the constraint before wiring user input to the API parameter.
